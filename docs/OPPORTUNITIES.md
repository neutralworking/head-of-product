# OPPORTUNITIES

Append-only log of monetisation opportunities surfaced for Luke's portfolio. HoP adds rows; Luke disposes them.

## Schema

| Field | Notes |
|---|---|
| Date | ISO `YYYY-MM-DD` when surfaced |
| Source | Where the signal came from (e.g. `discovery-scan`, `repo:chief-scout`, `web:competitor`, `luke`, `hop-inference`) |
| Opportunity | One sentence — the move, not the rationale |
| Fit | `high` / `medium` / `low` against current portfolio & skills |
| Effort | `quick` (<1 day) / `small` (1–3 days) / `medium` (1–2 weeks) / `large` (>2 weeks) |
| Est. revenue | Honest first guess with timeframe, e.g. `£0–500 / 3mo`, `£500–2k / year`. `?` if unknown. |
| Next step | One concrete action — small, testable |
| Status | `open` / `pursuing` / `parked` / `killed` / `done` |
| Disposition | Luke's call + date if not `open` |

## Rules

- Append only. Don't rewrite history; add a new row to flip status, referencing the original row's date+source.
- Lateral avenues (fractional PO, iGaming, etc.) **ask first** before logging — they touch agency reputation.
- Revenue estimates are guesses. £0 is a valid answer.
